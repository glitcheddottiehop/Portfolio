Introduction

Idea was to create a design for an app that could help out with organizing tasks. You could add tasks, make custom lists, set reminders, see schedule and track progress.

My Role

I had to create an app from scratch. I was given context but my task was to make a design using UI/UX principles, methods, trending features.

Problem

Idea is ready but it’s missing design which is where ideas start to materialize.

Color palette

I decided to use colors: white as primary, blue as secondary, green as tertiary and orange as accent. These 3 colors go well together. Color blue gives stability and reliability, green - refreshment and security. Orange - gives a little bit of fun, lighter mood.

Goals

  • To make app look trustworthy
  • To help organize tasks
  • To make app easy to use

Final Designs

Landing Page

When user opens app he sees all today’s tasks. If there is no tasks app shows “All done”.

With the plus sign on the bottom right corner user can add new list or new task.

Add task

User can add task by filling out fields like title and description. He can also add it to specific list.

Set date and time

User can set date/time reminder to a task. He can choose date, time and repeat.

Actions with tasks

When user has tasks in list he then can use swipe gestures to edit or delete task.

Create custom list

When user is creating a list he must provide list’s name and he can also immediately add task to it.

Menu

Menu has default lists like Today and Tomorrow. There is also status lists which shows tasks that are missed and done.

Schedule menu

When user taps on calendar icon app shows calendar with different views.

Graph

View graph shows all tasks, date and time.

Day

View day shows every hour and corresponding task to it.

Week

View week shows hours, dates and corresponding tasks.

Month

View month shows week days, dates and corresponding tasks.

Done list

User can see done tasks tapping on list Done in menu.

Chart

Chart shows progress for lists Today, Yesterday or custom date.

Filter by date

User can choose chart for custom date and select date range for chart.

Filtered results

When user applies filter he sees corresponding progress for selected date range.

Impact

The moment I gave some users to try out new design prototypes I received these feedbacks:

  • “I don’t think color orange goes well with green”
  • “I don’t like the font”
  • “There is no point of list Done”

Final thoughts

I learned that it’s not useful to create design for every possible screen size and it is overwhelming. Learned more about creating responsive screens. Tried out different animations. Wouldn’t call this my best project I see a lot of things to improve here for example - menus.
