Introduction

Idea was to create a design for an app that could help you out with deciding what to do in your free time. It takes into account such factors as mood, preferences and time

It revolves around question “What would you like to do?” and so is called the app.

My Role

I had to create an app from scratch. I was given context but my task was to make a design using UI/UX principles, methods, trending features.

Problem

Idea is ready but it’s missing design which is where ideas start to materialize.

Color palette

I chose to use color white as primary and color blue as secondary color. Color blue in color psychology is described like this “Blue calls to mind feelings of calmness and relaxation. It is often described as peaceful, tranquil, secure, and orderly. Blue is seen as a sign of stability and reliability. Businesses that want to project an image of security often utilize blue in their advertising and marketing efforts.”

Specifications

Technical specifications were: iOS at 375x812px resolution.

Target group

  • People who wants to try something new
  • Tourists. It could help them to get to know the city/town they’re visiting
  • People who has no time to search for activities themselves

Goals

  • To make app look professional and trustworthy
  • To make app easy to use
  • To make people lives easier and more interesting

Final Designs

Splash Screen

Decided to use the main question “What would you like to do?” on splash screen to seem that app is talking to you/asking this question every time user wants to figure out what he would like to do. As image I used group of people that are talking about something. Possibly they are talking about some activities to do?

Log in/Sign up

After question is asked it disappears and Welcome screen is giving user option to make a new account or sign up to already existing account.

Sign up

When creating account user can also use sign up options like “Sign up with Google” so user can skip sign up process.

Log in

After account is created user can log in using data for creating account.

Add mood

After successful log in/sign up user is prompted to add current mood to get personalized feed with events.

Energy level

After adding mood user is prompted to add energy level (how active user feels now) to get personalized feed with events.

Interests

After adding energy level user is prompted to add interests to get personalized feed with events. Interests shows up as “chips” or “pills” that user can easily remove with “x”.

To help user add interests quicker app suggests some popular topics.

Feed

After adding preferences successfully user get personalized feed based on previously inserted data.

Feed is table with events, it shows event name, working hours (based on current time) and distance from current location.

Map

User can switch from List view to Map view to see events happening nearby in map.

Friends

App has also feature “Friends” which allows user to edit friend’s name, delete friend or block friend.

Having “Friends“ allows to add them to user’s local events.

Profile

User’s profile has picture, name surname, username, bio which user can edit.

Profile options

When using app’s search it shows corresponding events or users. User can tap on other user icon and add user as a friend, see information about user, block user or hide from search.

Event page

When tapping on event from Feed it opens more information about the event. Such information like rating, price and description.

User can share event, add to favorites or see event on Map.

Favorited events

When event is favorited it is saved in “My events” where user can easily open event page or remove it from favorites.

User can also see search history.

Created events

When user has created event it is saved in “My events” under “Created”. User can also find created templates

Create event

User can make his own local event. He has to fill in information about event like name; location (where user can use current location or write in specific loaction); set event time from-to or set it to “All day”, user can also set it as recurring event; date; description; price; add friends.

Filter events

User can apply filter to events in feed.

He can filter by: interests, location, distance and price.

Account settings

Under “Account” user can find settings for account.

Edit moods

If users taps on “Edit moods” he can edit each mood. He can edit: emoji and tags to include/exclude.

Edit energy

User can also “Edit energy”. . He can edit: emoji and tags to include/exclude.

Edit interests

User can edit interests. App once again is suggesting popular topics to help user with adding inetersts.

Impact

The moment I gave some users to try out new design prototypes I received these feedbacks:

  • “Looks professional”
  • “Too much text, I don’t want to receive so much info”
  • “Too colorful - distracts me from what I should look at”

Final thoughts

It was a valuable experience. I learned more about iOS and screen sizes. I learned that too much animations and colors can not just get users attention but also distract user. I see that ability to making design systems is valuable as well. I will definitely continue to make sketches, user flows, design systems and my favorite part - prototypes!
